## Purpose

Evidence files provide verification that:
1. Documented features actually work as described
2. Test results can be reproduced
3. Claims in documentation have supporting data

## Adding New Evidence

When adding evidence:
1. Use date-stamped filenames (YYYY-MM-DD format)
2. Include context in commit message
3. Reference from relevant documentation
